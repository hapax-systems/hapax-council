# Doc Claims Carry Recheck Commands (always-on)

## Checklist

- [ ] recheck-cmds-present: Every operational claim added to docs/runbooks carries a recheck command.
- [ ] claims-match-code: Doc statements agree with the code in this diff — names, paths, ports, defaults.
- [ ] stale-docs-updated: Docs invalidated by this diff are updated in the same PR.

## Verdict contract

Address every checklist item explicitly as pass / finding / NA in your review output.
