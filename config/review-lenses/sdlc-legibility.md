# SDLC Legibility (reason-codes name truth)

## Checklist

- [ ] reason-codes-name-truth: Every new blocker/reason code names the actual failure, not a side symptom.
- [ ] no-euphemism: Failure paths log what actually happened — a confusion in the SDLC is a FAILURE of the SDLC.
- [ ] receipts-traceable: Receipts/dossiers record enough to reconstruct the decision later.

## Verdict contract

Address every checklist item explicitly as pass / finding / NA in your review output.
