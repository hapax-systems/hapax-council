# Codex Operator Dossier Ingress

Codex bootstraps include a sanitized operator dossier block generated by
`scripts/codex-operator-dossier-context.py`. The block is appended to generated
`scripts/hapax-codex` spawn files only when Codex is launched with a task or
terminal session.

The default source is `config/codex/operator-dossier-summary.md`. That file is
prompt-visible and must stay decision-predictive: autonomy heuristics, operating
preferences, governance constraints, and update rules only. It must not contain
secrets, copied private memory, raw transcripts, personal logs, or credential
values.

The private Claude dossier remains a pointer, not a prompt payload:
`~/.claude/projects/-home-hapax-projects/memory/user_profile.md`. Codex may use
the safe summary and current user/task/relay context; it must not copy that
private file into bootstrap material.

## Update

Update the safe summary when the operator gives a durable directive, corrects a
broad work-style assumption, or asks for a reviewed relay summary to become
Codex-visible. Keep changes short and prompt-auditable.

Do not update it with task-local facts, transient system state, one-off bug
fixes, or raw conversation excerpts. Those belong in task notes, relay YAML,
handoffs, or incident docs.

## Invalidation

Invalidate or downgrade the summary to pointer-only when an operator correction
contradicts it, when a source is discovered to include sensitive material, when
the private dossier pointer moves, or after a major workflow change that makes
the heuristics stale.

The renderer fails closed. If the configured source contains common secret
assignments, token-looking values, or raw dialogue markers, generated bootstraps
receive an `unavailable` block with a repair instruction and no copied source
content.
